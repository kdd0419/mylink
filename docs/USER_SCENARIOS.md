# 마이링크 (My Link) - 사용자 시나리오 (User Scenarios)

이 문서는 '마이링크' 서비스의 주요 사용자(소유자 및 방문자)가 겪는 핵심 시나리오를 정의합니다.

## 1. 소유자 (Owner) 시나리오

### 1.1 회원가입 및 프로필 초기화
- **사용자는 서비스를 시작하기 위해 "Google로 시작하기" 버튼을 클릭하여 로그인한다.**
    - *System*: Google 로그인 팝업을 띄우고 인증을 진행한다.
- **사용자는 서비스 이용을 시작하기 위해 Google 계정 연동을 승인한다.**
    - *System*:
        - 신규 유저: Google 프로필(이름, 이메일, 사진) 정보를 가져와 계정을 생성하고 초기 대시보드로 이동시킨다.
        - 기존 유저: 대시보드로 즉시 이동시킨다.
- **사용자는 별도의 아이디 설정 없이 본인의 Google 이메일 ID(@ 앞자리)를 기반으로 고유 URL(`mylink.com/username`)을 자동으로 할당받는다.**

### 1.2 링크 관리 (CRUD)
- **사용자는 자신의 SNS나 블로그를 연결하기 위해 "새 링크 추가" 버튼을 눌러 제목과 URL을 입력한다.**
- **사용자는 링크를 등록할 때 대상 URL의 파비콘(Favicon)을 자동으로 가져오도록 설정한다.** (별도 아이콘 설정 없음)
- **사용자는 오타를 고치거나 링크를 변경하기 위해 기존 링크 아이템의 링크 텍스트를 눌러 제목이나 URL을 인라인 편집으로 업데이트한다.**
- **사용자는 더 이상 필요 없는 링크를 제거하기 위해 링크 아이템의 "삭제" 버튼을 눌러 목록에서 완전히 삭제한다.**

### 1.3 프로필 관리 - 인라인 편집
- **사용자는 자신을 소개하기 위해 대시보드에서 닉네임과 한 줄 소개(Bio)를 편집한다.**

---

## 2. 방문자 (Visitor) 시나리오

### 2.1 페이지 방문 및 탐색
- **방문자는 소유자의 정보를 확인하기 위해 소유자의 고유 URL(`mylink.com/username`)로 접속한다.**
- **방문자는 소유자의 프로필 이미지, 닉네임, 그리고 한 줄 소개를 한눈에 확인한다.**
    - *System*: 유효한 URL인 경우 해당 사용자의 프로필(사진, 이름, 소개)과 링크 목록을 로드하여 보여준다.

### 2.2 링크 이용
- **방문자는 소유자의 다른 채널로 이동하기 위해 목록에 나열된 링크 중 원하는 버튼을 클릭한다.**
- **방문자는 링크 옆에 표시된 파비콘 아이콘을 보고 해당 링크가 어떤 서비스(GitHub, 블로그 등)인지 시각적으로 인지한다.**
    - *System*: 새 탭에서 해당 URL을 연다. (추후 이 시점에 클릭 이벤트를 카운팅한다.)
